# Track Analysis Tool

## Что это делает

Editor Tool анализирует `AudioClip` внутри Unity Editor и сохраняет результат в JSON для будущего visualizer/runtime.

## Как работает MVP

- читает сырой аудио-массив через `AudioClip.GetData`
- делит его на шаги анализа
- считает среднюю абсолютную амплитуду для каждого шага
- сглаживает значения простым скользящим окном
- находит грубые пики по порогу и локальному максимуму
- при включённом `Use Manual BPM` строит список beat times по формуле `60 / BPM`

## Что входит в JSON

- базовая информация о треке
- параметры анализа
- `peaks` с временем и силой
- `energyFrames` с временными срезами энергии
- `beatTimes`, если включён ручной BPM

## Что пока не реализовано

- auto BPM detection
- section detection
- beat/bar/phrase semantic markers
- сложный DSP-анализ и спектральные признаки

## Практические замечания

- Для корректного чтения данных Unity должен уметь отдавать sample data из клипа.
- Слишком маленький `Analysis Step` даст больше точек и более шумный результат.
- `Smoothing Window` помогает сгладить энергию.
- `Peak Threshold` лучше подбирать под конкретный трек.
